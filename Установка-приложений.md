# Установка приложений Messenger на устройства

Руководство для обычного пользователя: как поставить мессенджер на телефон,
компьютер или открыть его в браузере.

> **Перед началом:** у вас уже должен быть запущен сервер Messenger и его адрес
> (например `https://chat.example.com` или `http://localhost:8080`).
> Если сервера ещё нет — сначала прочитайте **`Установка-сервера.md`**.

---

## Какой клиент выбрать

| Устройство | Рекомендуемый способ | Нужна установка? |
|---|---|---|
| Любой браузер (Windows/macOS/Linux/Android/iOS) | **PWA (веб-версия)** | Нет — открывается сразу |
| Android-телефон | APK-файл | Да |
| Mac (Apple Silicon) | Приложение `.dmg` / `.app` | Да |
| Windows / Linux (десктоп) | Нативный пакет (`.msi` / `.deb`) или JAR | Да |
| iPhone / iPad | PWA или сборка через Xcode | См. раздел iOS |

**Самый простой путь для всех — PWA.** Нативные приложения нужны, только если вы
хотите отдельную иконку, фоновые push и интеграцию с ОС.

---

## 1. PWA — веб-версия (проще всего, работает везде)

Веб-клиент уже встроен в сервер. Отдельно ставить ничего не нужно.

**Шаг 1.** Откройте адрес сервера в браузере:
```
http://localhost:8080         (локально)
https://chat.example.com      (если настроен домен)
```

**Шаг 2.** Установите как приложение (необязательно, но удобно — будет своя иконка):
- **Chrome / Edge** (Windows, macOS, Linux, Android): значок установки в адресной строке → «Установить».
- **Safari (iPhone/iPad):** кнопка «Поделиться» → «На экран „Домой“».
- **Safari (macOS):** меню «Файл» → «Добавить в Dock».

Готово — мессенджер открывается как обычное приложение.

---

## 2. Android (файл APK)

**Готовый файл:** `apps/mobile/android/build/outputs/apk/debug/messenger-android-debug.apk`

**Шаг 1.** Перенесите этот APK на телефон (через кабель, облако или мессенджер).

**Шаг 2.** На телефоне откройте файл. При первой установке Android попросит
разрешить установку из этого источника — нажмите «Разрешить» / «Настройки» →
включите для вашего файлового менеджера или браузера.

**Шаг 3.** Установите и откройте приложение. При первом запуске введите адрес
вашего сервера (см. раздел «Адрес сервера» ниже).

**Собрать самому** (если нужен свежий или подписанный release-APK):
```bash
chmod +x install-client-android.sh
./install-client-android.sh        # macOS / Linux
install-client-android.bat         # Windows
```
Требуется JDK 17+. Для автоустановки на подключённое устройство — Android SDK/adb.

## 3. Mac (Apple Silicon)

**Готовые файлы:**
- `mess_client_macos/Messenger.dmg` — образ для установки (94 МБ);
- `mess_client_macos/Messenger.app` — само приложение.

**Шаг 1.** Откройте `Messenger.dmg` двойным щелчком.

**Шаг 2.** Перетащите `Messenger.app` в папку «Программы».

**Шаг 3.** Запустите. Поскольку приложение не из App Store, при первом запуске
macOS может предупредить о неизвестном разработчике — нажмите правой кнопкой по
иконке → «Открыть» → «Открыть» (или разрешите в «Системные настройки →
Конфиденциальность и безопасность»).

> Этот сборка — под **Apple Silicon (M1/M2/M3)**. Для Intel-Mac соберите из
> исходников (см. раздел 4, вариант JAR) или через CI.

## 4. Windows и Linux (десктоп)

Для десктопа есть два пути.

### Вариант A. Кроссплатформенный JAR (быстро, нужна Java)

**Готовый файл:** `apps/desktop/build/libs/messenger-desktop.jar`

Установите Java 17+ ([adoptium.net](https://adoptium.net/)) и запустите:
```bash
java -jar apps/desktop/build/libs/messenger-desktop.jar
```
Работает на Windows, Linux и Intel-Mac одинаково.

### Вариант B. Нативный установщик (`.msi` / `.deb`)

Нативные пакеты собираются **на целевой ОС** (Windows MSI — только на Windows,
Linux DEB — только на Linux). Это ограничение инструмента `jpackage`.

Собрать на нужной ОС:
```bash
cd apps/desktop
./gradlew packageMsi     # Windows  → build/compose/binaries/main/msi/
./gradlew packageDeb     # Linux    → build/compose/binaries/main/deb/
./gradlew packageDmg     # macOS    → build/compose/binaries/main/dmg/
```

Либо собрать всё сразу для всех платформ автоматически — через GitHub Actions
(см. раздел 6).

## 5. iPhone / iPad (iOS)

Готового `.ipa` нет: для установки нативного iOS-приложения нужен **Mac с Xcode** и
**аккаунт Apple Developer** (требование Apple, обойти нельзя).

Варианты:
1. **PWA (рекомендуется для большинства):** откройте сайт сервера в Safari →
   «Поделиться» → «На экран „Домой“». Это бесплатно и без Xcode.
2. **Нативная сборка:** откройте проект `apps/mobile/ios/` в Xcode, укажите свою
   команду разработчика (signing), соберите и установите на устройство или
   опубликуйте через TestFlight.

## 6. Где взять готовые файлы для всех платформ сразу

Штатный способ собрать **DMG / MSI / DEB / APK** для всех платформ одновременно —
workflow **`build-native.yml`** в GitHub Actions:

- запускается автоматически при создании тега версии (`git tag v1.0.0 && git push --tags`);
- или вручную: вкладка **Actions → Build Native Clients → Run workflow** (можно
  указать версию и адрес сервера для «вшивания»).

Готовые файлы появятся в артефактах сборки и в разделе Releases. Адрес сервера
можно «вшить» при сборке (переменная `SERVER_URL`), тогда пользователю не придётся
вводить его вручную.

---

## 7. Адрес сервера в приложении

- **PWA / веб:** адрес определяется автоматически — это адрес, по которому вы
  открыли сайт. Ничего вводить не нужно.
- **Нативные приложения (Android / Desktop / iOS):** при первом запуске откроется
  экран настройки сервера — введите полный адрес, например `https://chat.example.com`.
  Если адрес «вшит» при сборке через CI — этот экран можно пропустить.

После ввода адреса войдите под своим логином/паролем (или зарегистрируйтесь, если
сервер это разрешает — см. режим регистрации в `Установка-сервера.md`).

## 8. Сводка готовых файлов в репозитории

| Платформа | Файл | Статус |
|---|---|---|
| Веб / PWA | встроена в сервер (`client/dist/`) | ✅ готово |
| Android | `apps/mobile/android/build/outputs/apk/debug/messenger-android-debug.apk` | ✅ готово (debug) |
| macOS (Apple Silicon) | `mess_client_macos/Messenger.dmg` и `Messenger.app` | ✅ готово |
| Desktop (любая ОС) | `apps/desktop/build/libs/messenger-desktop.jar` | ✅ готово (нужна Java 17+) |
| Windows `.msi` / Linux `.deb` | — | ⚙️ собрать на целевой ОС или через CI |
| iOS `.ipa` | — | ⚙️ нужен Xcode + Apple Developer |

---

## 9. Если что-то не работает

| Симптом | Решение |
|---|---|
| Android: «Установка заблокирована» | Разрешите установку из этого источника в настройках |
| macOS: «Приложение нельзя открыть» | ПКМ по иконке → «Открыть», либо «Системные настройки → Конфиденциальность» |
| Десктоп-JAR не запускается | Установите Java 17+ и проверьте `java -version` |
| Приложение не подключается к серверу | Проверьте адрес сервера и что сервер доступен (откройте его в браузере) |
| Не приходят push-уведомления | Убедитесь, что на сервере сохранены VAPID-ключи (см. `Установка-сервера.md`) |

# Установка клиента Messenger на Windows 11

## Требование: сервер Messenger уже должен быть запущен

Прежде чем устанавливать клиент, убедитесь, что сервер доступен.  
Откройте в браузере `http://адрес-сервера:8080` — должна открыться страница входа.

---

## Вариант А — PWA в браузере ✅ Рекомендуется

**Самый простой способ.** Не требует установки дополнительных программ.  
Работает как полноценное приложение после «установки» в браузере.

### Шаг 1. Откройте сервер в браузере

Запустите **Microsoft Edge** или **Google Chrome** и откройте адрес сервера:

```
http://localhost:8080          (если сервер на этом же компьютере)
https://chat.example.com       (если сервер доступен по домену)
```

### Шаг 2. Войдите или зарегистрируйтесь

- Если у вас уже есть аккаунт — войдите
- Если регистрация открыта — создайте новый аккаунт
- Если регистрация по инвайту — попросите код у администратора

### Шаг 3. Установите как приложение (опционально)

**Microsoft Edge:**
1. В правом верхнем углу нажмите `...` (три точки)
2. «Приложения» → «Установить этот сайт как приложение»
3. Нажмите «Установить»

**Google Chrome:**
1. В адресной строке появится значок установки (экран со стрелкой ↑)
2. Нажмите на него → «Установить»

После установки Messenger появится в меню «Пуск» как обычное приложение.

---

> **Важно для PWA:** Если сервер доступен только по HTTP (не HTTPS), установка как  
> PWA-приложения не будет работать — только localhost является исключением.  
> Для доступа извне настройте HTTPS на сервере (TLS-сертификат или Cloudflare Tunnel).

## Вариант Б — Desktop-приложение (Compose Multiplatform)

Нативное Windows-приложение. Требует JDK 17+.

### Шаг 1. Установите JDK 17+

1. Перейдите на https://adoptium.net/
2. Скачайте **Temurin 17** (LTS) для Windows x64
3. Запустите установщик — оставьте все настройки по умолчанию
4. При появлении вопроса **«Add to PATH»** — выберите «Entire feature will be installed»

Проверьте установку в новом терминале:

```powershell
java -version
# должно показать: openjdk version "17.x.x" ...
```

### Шаг 2. Получите исходный код

Если у вас ещё нет репозитория:

```powershell
git clone <URL_репозитория> messenger
cd messenger
```

### Шаг 3. Запустите сборку

```bat
install-client-windows.bat
```

Скрипт:
1. Проверит наличие JDK
2. Соберёт приложение через Gradle (`gradlew.bat packageMsi`)
3. Предложит запустить MSI-установщик

Первая сборка занимает **5–10 минут** (скачиваются зависимости Gradle и Kotlin).  
Последующие сборки — 1–2 минуты.

### Шаг 4. Установите приложение

Скрипт найдёт MSI-файл в `apps\desktop\build\compose\binaries\main\` и предложит его запустить.

Или запустите вручную:

```powershell
# Найти MSI
Get-ChildItem apps\desktop\build\compose\binaries\main -Filter *.msi

# Запустить
Start-Process "apps\desktop\build\compose\binaries\main\Messenger-1.0.msi"
```

### Шаг 5. Первый запуск

1. Откройте **Messenger** из меню «Пуск»
2. На экране настройки введите URL вашего сервера:
   ```
   http://localhost:8080
   ```
   или если сервер на другом компьютере:
   ```
   https://chat.example.com
   ```
3. Войдите или зарегистрируйтесь

## Сравнение вариантов

| | PWA (браузер) | Desktop-приложение |
|--|--|--|
| Установка | Мгновенно | ~10 мин (сборка) |
| Требования | Только браузер | JDK 17+, Git |

## Устранение типичных проблем

### PWA не открывается

Проверьте, что сервер запущен:

```powershell
sc query Messenger       # если установлен через нативный установщик
# или
docker compose ps        # если через Docker
```

### JDK не найден при сборке Desktop-клиента

```powershell
# Найти установленные версии Java
Get-Command java -ErrorAction SilentlyContinue
[System.Environment]::GetEnvironmentVariable("JAVA_HOME", "Machine")
```

После установки JDK закройте все окна терминала и откройте новое.

### Gradle не загружает зависимости (корпоративная сеть)

Добавьте настройки прокси в файл `%USERPROFILE%\.gradle\gradle.properties`:

```properties
systemProp.http.proxyHost=proxy.example.com
systemProp.http.proxyPort=8080
systemProp.https.proxyHost=proxy.example.com
systemProp.https.proxyPort=8080
```

### Desktop-приложение не подключается к серверу

1. Убедитесь, что URL сервера введён без лишних символов (без `/` в конце)
2. Проверьте брандмауэр: порт сервера должен быть доступен
3. Если сервер на другом компьютере, убедитесь, что указываете его IP или домен, а не `localhost`

### Ошибка «The certificate is not trusted» при HTTPS

Если на сервере используется самоподписанный сертификат:
- **PWA:** В браузере нажмите «Дополнительно» → «Перейти на сайт»
- **Desktop:** Добавьте сертификат в доверенные (`certmgr.msc`)

Для продакшн-сервера используйте сертификат от Let's Encrypt или Cloudflare Tunnel.

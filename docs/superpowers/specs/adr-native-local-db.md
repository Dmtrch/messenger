# ADR: Native Local Database

**Статус:** Accepted  
**Дата:** 2026-04-11  
**Контекст:** Foundation для нативных клиентов Messenger

---

## Контекст

Текущий PWA-клиент уже определил продуктовую offline-модель:

- локальный кэш истории;
- оффлайн-просмотр;
- outbox;
- retry после восстановления сети;
- cursor-based догрузка более старой истории;
- локальное состояние чатов и сообщений.

Для нативных клиентов нужно повторить эту модель без browser runtime.

---

## Решение

Нативным стандартом локальной БД фиксируется `SQLite`.

Важно:

- это не попытка повторить браузерный storage буквально;
- это перенос той же продуктовой semantics PWA-клиента на нативный runtime;
- `SQLite` выбирается как базовая локальная БД для `Desktop`, `Android`, `iOS`.

---

## Причины решения

`SQLite` лучше всего совпадает с текущими потребностями проекта:

- подходит для message history и chat list;
- подходит для outbox/sync queue;
- хорошо поддерживает cursor-based pagination и индексацию;
- доступна на всех целевых платформах;
- не требует разных ментальных моделей хранения по платформам.

---

## Отложенные вопросы

Следующие решения не принимаются на этапе Foundation:

- использовать ли `SQLCipher` для шифрования локальной БД;
- нужен ли отдельный encrypted profile layer поверх SQLite;
- будет ли единая ORM/driver abstraction для всех нативных клиентов.

Если эти вопросы понадобятся, они выносятся в отдельные ADR.

---

## Последствия

Плюсы:

- единая storage semantics для всех платформ;
- проще переносить offline/sync модель из PWA;
- легче поддерживать cross-platform test cases.

Минусы:

- потребуется platform-specific database integration;
- локальное шифрование БД остаётся отдельным следующим решением, а не частью Foundation.

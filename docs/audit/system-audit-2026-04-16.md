## 2. Анализ реализации по компонентам

### Сервер (Backend)
- **Архитектура:** Чистая реализация на Go с разделением по модулям (auth, chat, keys, media, ws).
- **Безопасность:** Реализован cookie-based refresh token, проверка прав доступа к чатам, хранение хэшей паролей (bcrypt).
- **E2EE Support:** Сервер выступает как доверенное хранилище PreKey Bundle (X3DH), не имея доступа к ключам шифрования.
- **Admin:** Полный набор API для управления заявками на регистрацию и инвайт-кодами.

### Web-клиент (React + Vite)
- **UI/UX:** Наиболее зрелая платформа. Реализованы все основные экраны, включая администрирование.
- **Криптография:** Полная поддержка Double Ratchet и X3DH в браузере (через `WebCrypto` и `Sodium`).
- **Звонки:** Рабочий `CallOverlay` с поддержкой WebRTC через WebSocket-сигнализацию.

### Desktop & Android (Compose Multiplatform)
- **UI:** Используется общий код для большинства экранов (Compose).
- **Криптография:** Добавлен `SessionManager.kt` (реализация Double Ratchet на Kotlin).
- **Пробелы:** 
    - Отсутствует UI для поиска пользователей (`/api/users/search`) и создания чатов.
    - Требуется интеграция `SessionManager` в `WSOrchestrator` для автоматического шифрования исходящих сообщений.

### iOS (SwiftUI)
- **UI:** Нативный интерфейс, соответствующий гайдлайнам Apple.
- **Критический риск:** 
    - **Отсутствует E2EE менеджер.** Код не умеет обрабатывать X3DH рукопожатия.
    - **Неверный транспорт.** Клиент пытается отправлять сообщения через REST `POST /api/messages`, который не поддерживается сервером (сервер ждет сообщения через WebSocket).

## 3. Выявленные блокирующие проблемы (Roadmap)

1.  **[High] Унификация транспорта:** Перевести iOS-клиент на отправку сообщений через WebSocket (синхронизация с Web/Android).
2.  **[High] iOS E2EE:** Портировать логику `SessionManager` на Swift для поддержки зашифрованных чатов.
3.  **[Medium] New Chat UI:** Добавить во все нативные клиенты экран поиска пользователей и создания диалогов.
4.  **[Medium] WebRTC Stability:** Доработать `CallOverlay` на мобильных платформах для стабильной работы видеозвонков.
5.  **[Low] Улучшение безопасности:** Переход на `EncryptedSharedPreferences` (Android) и `Keychain` (iOS) для хранения ключей.

---

## 4. Вердикт аудита
Система готова к эксплуатации в режиме **Self-hosted Web Messenger**. Нативные приложения находятся в состоянии "UI-готовности", но требуют завершения интеграции криптографического слоя и исправления транспортных протоколов для полной совместимости с сервером.

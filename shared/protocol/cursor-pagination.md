# Cursor-Based Pagination Contract

`Cursor-based pagination` — mandatory capability для всех клиентов Messenger:

- `Desktop`
- `Android`
- `iOS`
- web/PWA

## Правило

Ни один клиент не должен ограничиваться только локальным кэшем старых сообщений. Каждый клиент обязан уметь догружать более старую историю с сервера по cursor-based модели.

## Каноническая модель

Базовая текущая semantics:

- сервер возвращает страницу сообщений;
- `nextCursor` указывает на следующую порцию старой истории;
- cursor считается opaque contract;
- клиент повторяет запрос до исчерпания истории.

## Что считается обязательным

1. Получение первой страницы истории чата.
2. Догрузка старой истории по cursor.
3. Совместная работа pagination и offline cache.
4. Совместная работа pagination и multi-device состояния.

## Что считается недопустимым

- local-only infinite scroll без серверной догрузки;
- pagination по page number для нативных клиентов как отдельная ветка протокола;
- разные правила окончания истории на разных клиентах.
